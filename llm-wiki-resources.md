# llm-wiki Resources

A curated collection of blog posts, videos, and industry reactions to Andrej Karpathy's **llm-wiki** idea — using LLM agents to build and maintain a persistent, evolving Markdown knowledge base.

---

## The Original Gist

- **[llm-wiki — Andrej Karpathy](https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f)**
  The original GitHub Gist where Karpathy outlined the llm-wiki concept: an LLM agent that iteratively reads, writes, cross-links, and curates structured Markdown documents as a compounding personal knowledge base. This contrasts with stateless RAG setups by keeping knowledge persistent, auditable, and ever-growing.

## Key Themes

| Theme | Summary |
|---|---|
| **Persistent knowledge** | Unlike RAG, llm-wiki keeps knowledge in living Markdown files that grow and cross-link over time. |
| **Agent-driven curation** | An LLM agent reads, writes, and refines entries — compounding knowledge rather than just retrieving it. |
| **Beyond RAG** | The pattern avoids the stateless, context-window-limited nature of RAG for domains where continuity matters. |
| **Vibe coding connection** | llm-wiki fits naturally into the "vibe coding" paradigm where LLMs are AI-native development and knowledge partners. |
